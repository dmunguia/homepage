Programación Orientada a Objetos

Objetos

desde el paradigma de OO se considera que los objetos se componen de:

  • propiedades y/o estado
  • comportamiento (métodos)

modelar el concepto de automóvil como un objeto

dependiendo del contexto del problema computacional que se quiera modelar, las propiedades del objeto modelado pueden variar categóricamente

perspectivas: técnica y comercial

ejemplo

modelar figuras geométricas como clases

principios teóricos

  • abstracción
  • encapsulamiento
  • modularidad
  • jerarquía

abstracción

  • al describir o modelar un fenómeno particular se incluyen o especifican aspectos considerados relevantes dentro del contexto que se realiza la abstracción, a la misma vez, se descartan otros detalles considerados no esenciales
  • desde una perspectiva funcional permite la re-utilización de código: los desarrolladores pueden implementar/probar/depurar objetos que realizan tareas específicas, que posteriormente pueden ser utilizados en el código propio con toda confianza (bibliotecas de objetos)

encapsulamiento

  • ocultamiento de información: al interactuar únicamente con los métodos de un objeto, lo detalles de la implementación del objeto permanecen ocultos del mundo exterior
  • garantiza la integridad de los datos del objeto, pues la única manera de modificarlos es a través de las reglas definidas en sus métodos

modularidad

  • el código de un objeto puede ser escrito y mantenido independientemente del código de otros objetos, una vez creado el objeto puede ser fácilmente utilizado por diferentes aspectos de la aplicación
  • concepto de caja de herramientas

import math

CIRCULO = 0
TRIANGULO = 1
CUADRADO = 2
RECTANGULO = 3
def calcular_area(tipo_figura, lado, radio, largo, ancho):
	if (tipo_figura == CIRCULO):
		area = math.pi * radio ** 2
	elif (tipo_figura == TRIANGULO):
		area = lado ** 2 * (math.sqrt(3) / 4)
	elif (tipo_figura == CUADRADO):
		area = lado ** 2
	elif (tipo_figura == RECTANGULO):
		area = largo * ancho / 2

	return area

jerarquía

  • los objetos podrían tener cierta cantidad de características (estado, propiedades y comportamiento) en común
  • a la vez, los mismos objetos podrían definir, cada uno, características particulares que los diferencien de los otros objetos
  • la programación orientada a objetos permite a los objetos heredar propiedades y métodos de otras clases de objetos, formando así una jerarquía

mensajes dinámicos (dynamic dispatch)

  • las llamadas a métodos se conocen también como intercambio de mensajes entre objetos
  • mensajes dinámicos: se refiere a la capacidad de la plataforma para determinar en tiempo de corrida cuál es el método exacto que debe ser invocado según la jerarquía
  • permite a una nueva versión de una abstracción (clase) desplegar nuevo comportamiento refinado, incluso cuando es utilizada en un contexto que espera una versión anterior

tipos de lenguajes OO

  • basados en clases (Java, Smalltalk, Python)
  • basados en prototipos (Javascript, ActionScript, Lua)

OO basada en clases

  • clases: definen el comportamiento y la estructura del objeto
  • instancias: objetos utilizables, se crean a partir de las clases y contienen los datos a utilizar

OO basada en prototipos

  • no hay clases, sólo objetos
  • se pueden crear objetos vacíos (ex-nihilo) que luego son refinados con datos y comportamiento, o se pueden clonar objetos pre-existentes (el objeto original se convierte en prototipo del nuevo objeto)
  • la creación de un nuevo objeto a partir la clonación de otro ya existente es una forma de herencia

> yaris = {
	placa: 463076, marca: "toyota", modelo: "yaris", 
	toString: function() { return this.marca + ' ' + this.modelo; } 
}
  Object {placa: 463076, marca: "toyota", modelo: "yaris", toString: function}
> yaris.toString()
  "toyota yaris"
> yaris_del_año = {
	año: 2013, 
	toString: function() { return this.marca + ' ' + this.modelo + ' ' + this.año; } 
}
  Object {año: 2013, toString: function}
> yaris_del_año.toString()
"undefined undefined 2013"
> yaris_del_año.__proto__ = yaris
  Object {placa: 463076, marca: "toyota", modelo: "yaris", toString: function}
> yaris.isPrototypeOf(yaris_del_año);
  true
> yaris_del_año.toString()
  "toyota yaris 2013"
> yaris.marca
  "toyota"
> yaris_del_año.marca
  "toyota"
> yaris.año
  undefined
> yaris_del_año.año
  2013
